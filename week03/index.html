Chapter 5 - Objects

  • When creating an object literal, you can use square brackets to create a computed property name. JS expressions can also be used as the value of a property.
  • You can use the in operator to check if a property exists on an object. in can also be used to loop through the keys of an object.
  • You can also use the hasOwnProperty() method, which will check if the object has a property that has not been inherited.
  • Objects can be passed as parameters to functions. The function can use destructuring to choose which properties are used in the function.
  • The this keyword refers to the object it is inside of. It can be used to allow functions inside of an object to access other properties of the object.

Chapter 6 - The DOM

  • The childNodes property of an element returns a NodeList of all the child nodes of that element, including text nodes. The children property of an element returns a HTMLCollection of all the child elements of that element, excluding text nodes. There are also firstChild, lastChild, nextSibling, and previousSibling properties.
  • Node lists are not arrays, but they can be converted to arrays using the Array.from() method, or the spread operator.
  • A DOM node can only exist in one place in the DOM. Using the appendChild() or insertBefore() methods will move the node to a different location in the DOM.

Chapter 7 - Events

  • When handling keyboard events, you can check for keyboard combinations using the event's shiftKey, ctrlKey, and altKey properties.
  • Click events go through the capturing and bubbling phases. Capturing starts from outside in, and bubbling goes from inside out.
  • You can use the preventDefault() method to prevent the default action of an event.
  • You can use the stopPropagation() method to prevent the event from bubbling up the DOM tree.
  • Event delegation allows you to attach an event listener to a parent element that will handle events for any of its child elements.
  • The target property of an event object refers to the element that triggered the event (child element). The currentTarget property refers to the element that is listening for the event (parent element).
