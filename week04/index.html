Chapter 8 - Forms

  • Form input elements have multiple events, such as "focus" for when the input gains focus, "blur" for when the input loses focus, and "change" for when the value of the input changes.
  • The autofocus attribute can be used to set the focus to an input automatically when the page loads.
  • The preventDefault() function can be used to stop the form from being submitted.
  • The required attribute can be used to make an input required.
  • Forms provide several types of validation by default, but you can implement your own validation by modifying the submit event on the form.
  • The disabled attribute can be used to disable an input or a submit button.

Chapter 12 - OOP

  • Previously, a constructor function was needed to create objects. ES6 introduced the class declaration keyword, which can be used to create classes in a way that's similar to other OOP languages.
  • In both cases, the new keyword is used to create an instance of a class.
  • Classes can use static methods which belong to the class itself, and not to a specific instance.
  • The prototype of a class can be modified after the class has been declared to add new methods or properties. (Note: the prototype is "live", so any changes made to the prototype will be applied to all instances of the class, even if they have already been created.)
  • The hasOwnProperty() method can be used to check if a property belongs to the object or if it's inherited from the prototype.
  • Modifying the prototype of an object is risky, since you never know if a method will be added in the future. One solution is to create your own subclass of the object using the extends keyword, and then modify the subclass.
  • Using the Object.defineProperty() method, you can define a getter and setter for a property, which gives you greater control over how the property is assigned.
  • If you want to avoid using classes, you can use the Object.create() method to create a new object that inherits from the prototype of another object.
  • Returning this makes it possible to chain methods together.

Chapter 15 - Modular JavaScript

  • JavaScript modules have their own global scope, so the only methods that can be called are those that are exported using the export keyword.
  • JavaScript modules can be imported using the import keyword.
  • Node.js modules (also called Common JS modules) have been around longer and use a slightly different syntax than the new ES6 modules.
