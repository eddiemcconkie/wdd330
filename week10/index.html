MDN: Validating Forms

  • Client-side validation can provide a better user experience, but server-side security checks should always be performed on data. You should never trust data passed to your server from a client.
  • Attributes such as required, pattern, minlength and maxlength can be added to an input element to validate the data.
  • Form inputs can be styled using CSS pseudo-classes including :required, :valid and :invalid.
  • The Constraint Validation API provides forms and form elements with properties and methods for use in custom validation.
  • You can set the novalidate attribute on a form to prevent the browser's default validation, allowing you to customize validation.

MDN: Using Fetch

  • The fetch API is often used for simple GET requests, but can also be used for any HTTP request.
  • Headers are used for sending additional information with the request.
  • Response objects are returned from the fetch method and contain the response data and metadata.
  • The Body contains data in the form of JSON, text, FormData, Blob, ArrayBuffer, etc. and can be accessed using methods from the Request and Response interfaces.
