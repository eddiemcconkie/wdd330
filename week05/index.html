Chapter 10 - Testing and Debugging

  • Strict mode is a feature of JavaScript that prevents you from writing sloppy code. It catches additional errors and throws exceptions. Writing "use strict"; at the top of your file will enable strict mode. Strict mode is enabled by default in ES6 modules.
  • Feature Detection is a way to check if a browser supports a certain feature. For example, 
      if ("IntersectionObserver" in window) {...}
    will check if the browser supports the IntersectionObserver API.
  • console.log() or alert() can get help you solve simple bugs. However, learning to use the devtools in your code editor or browser will help you debug more complex problems.
  • You can use the debugger keyword to create a breakpoint in your code.
  • You can throw an error in your code using throw new Error("message"). Errors are caught in the try/catch block.
  • Testing libraries like jest can help you write tests and perform Test-Driven Development (TDD).
